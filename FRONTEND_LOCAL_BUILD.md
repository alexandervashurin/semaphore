# 📝 Изменения для локальной сборки фронтенда

## Проблема

Ранее фронтенд требовал постоянной загрузки ресурсов из интернета (CDN для шрифтов, иконок и т.д.), а также установки Node.js/npm на сервере.

## Решение

1. **Все ресурсы теперь локальные** - не требуют подключения к интернету
2. **Сборка через Docker** - не требует установки Node.js на сервере

## Внесенные изменения

### 1. Конфигурация Vue (`web/vue.config.js`)

**Изменено:**
- `outputDir` изменен с `../api/public` на `../web/public`
- Добавлена настройка `filenameHashing: false` для предсказуемых имен файлов
- Настроено именование JS/CSS файлов (`app.js`, `app.css`)

**Результат:** Vue-приложение компилируется непосредственно в `web/public/`, заменяя Vanilla JS версию.

### 2. Скрипты сборки (`web/package.json`)

**Добавлено:**
- Скрипт `build:clean` для чистой сборки

### 3. Docker для сборки

**Создано:**
- `web/Dockerfile.build` - Dockerfile для сборки
- `web/docker-compose.build.yml` - Docker Compose конфигурация
- `web/build.sh` - скрипт сборки через Docker

**Преимущества Docker-сборки:**
- ✅ Не требует Node.js на сервере
- ✅ Изолированная среда
- ✅ Воспроизводимые результаты
- ✅ Чистая система после сборки

### 4. Задачи сборки (`Taskfile.yml`)

**Добавлено:**
- `build:frontend` - сборка через Docker
- `build:frontend:local` - сборка через Node.js (для разработки)

### 5. Документация

**Обновлено:**
- `README.md` - обновлена секция установки
- `web/README_BUILD.md` - полное руководство по сборке

**Создано:**
- `FRONTEND_LOCAL_BUILD.md` - этот документ

## Локальные ресурсы

### Шрифты
- **Roboto** (все начертания): `web/src/assets/fonts/*.ttf`
  - Thin (100)
  - Light (300)
  - Regular (400)
  - Medium (500)
  - Bold (700)
  - Black (900)

### Иконки
- **Material Design Icons**: через npm-пакет `@mdi/font`
- **Кастомные иконки**: Vue-компоненты
  - OpenTofu
  - Pulumi
  - Terragrunt
  - HashiCorp Vault
  - DVLS

### Флаги
- 15 языковых флагов в `web/public/flags/`

## Инструкция по сборке

### Быстрая сборка

```bash
# Сборка всего проекта (frontend + backend)
task build
```

### Пошаговая сборка

```bash
# 1. Сборка frontend
cd web
npm install
npm run build

# 2. Сборка backend
cd ../rust
cargo build --release
```

### Запуск

```bash
# Из корня проекта
cd rust
cargo run -- server
```

Фронтенд будет доступен на `http://localhost:3000`

## Проверка

После сборки проверьте:

```bash
# Проверка файлов фронтенда
ls -lh web/public/

# Ожидаемые файлы:
# - index.html (Vue-приложение)
# - app.js (~500-800 KB)
# - app.css (~100-150 KB)
# - flags/ ( directory with SVG flags)
# - favicon.png
# - favicon.svg
```

## Преимущества

1. **Независимость от интернета** - все ресурсы локальные
2. **Быстрая загрузка** - нет задержек на загрузку из CDN
3. **Безопасность** - нет внешних зависимостей во время выполнения
4. **Работа в изолированных сетях** - не требуется доступ к внешним ресурсам
5. **Предсказуемость** - версии ресурсов фиксированы

## Размер бандла

- **JavaScript**: ~800 KB - 1.2 MB (сжатый)
- **CSS**: ~100-150 KB (сжатый)
- **Итого**: ~1-1.5 MB

## Требования

- **Node.js**: 16.x или новее
- **npm**: 7.x или новее
- **Rust**: 1.75 или новее (для backend)

## Миграция со старой версии

Если у вас уже установлена Semaphore:

```bash
# 1. Очистите старые файлы
rm -rf web/public/*.js web/public/*.css web/public/js

# 2. Установите зависимости
cd web
npm install

# 3. Соберите новую версию
npm run build

# 4. Проверьте работу
cd ../rust
cargo run -- server
```
