# Выполнение задач

> Как задачи ставятся в очередь, запускаются и логируются
>
> 📖 См. также: [Обзор системы](./system-overview.md), [Схема базы данных](./database-schema.md), [WebSocket API](../api-reference/websocket-api.md)

---

## Жизненный цикл задачи

```
Создание → Ожидание → Выполнение → Завершение (успех/провал)
```

1. **Создание** — пользователь или расписание запускает шаблон
2. **Очередь** — задача попадает в TaskPool
3. **Выполнение** — runner запускает Ansible/Terraform/скрипт
4. **Логирование** — вывод передаётся через WebSocket в реальном времени
5. **Завершение** — статус сохраняется в БД

## TaskPool

TaskPool — очередь задач с поддержкой параллельного выполнения.

- Настройка максимального параллелизма
- Приоритизация задач
- Отмена запущенных задач

## Runner

Runner — процесс, выполняющий задачи. Может работать локально или удалённо.

- Локальный runner (встроенный)
- Удалённый runner (отдельный процесс)
- HA-режим через Redis

## Логи

- Логи сохраняются в таблице `task_output`
- Потоковая передача через WebSocket
- Доступ через API: `/api/tasks/{id}/output`
