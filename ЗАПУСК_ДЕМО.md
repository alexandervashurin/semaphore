# 🚀 ЗАПУСК CRUD ДЕМО - ПОШАГОВАЯ ИНСТРУКЦИЯ

## ⚡ Быстрый старт (3 команды)

```bash
# 1. Запуск PostgreSQL и Frontend
./demo-start.sh

# 2. Запуск Backend (откройте НОВЫЙ терминал!)
./demo-start.sh --backend

# 3. Откройте браузер
# http://localhost/demo-crud.html
```

## 📋 Подробная инструкция

### Шаг 1: Проверка требований

```bash
# Проверьте наличие Docker
docker --version
docker-compose --version

# Проверьте наличие Rust (для backend)
cargo --version
```

Если что-то отсутствует:
- Установите Docker: https://docs.docker.com/get-docker/
- Установите Rust: https://rustup.rs/

### Шаг 2: Запуск баз данных и frontend

```bash
./demo-start.sh
```

Вы увидите:
```
✅ Docker и docker-compose найдены
ℹ  Запуск PostgreSQL и Frontend...
✅ PostgreSQL готов
✅ Docker сервисы запущены
```

**Не закрывайте терминал!** Docker контейнеры должны работать.

### Шаг 3: Запуск backend

**ОТКРОЙТЕ НОВЫЙ ТЕРМИНАЛ** и выполните:

```bash
./demo-start.sh --backend
```

Вы увидите:
```
✅ Rust/Cargo найден
ℹ  Запуск Rust backend...
ℹ  Сборка backend...
Server started at http://0.0.0.0:3000/
```

**Не закрывайте этот терминал!** Backend должен работать.

### Шаг 4: Откройте демо в браузере

Перейдите по адресу: **http://localhost/demo-crud.html**

### Шаг 5: Вход в систему

## 🐛 Решение проблем

### Проблема: "Address already in use"

**Решение:**
```bash
# Остановить процессы на порту 3000
pkill -f "semaphore server"

# Или через lsof
lsof -ti:3000 | xargs kill -9
```

### Проблема: "Docker не найден"

**Решение:**
```bash
# Установите Docker
# Для Ubuntu/Debian:
sudo apt-get update
sudo apt-get install docker.io docker-compose

# Для macOS: установите Docker Desktop
# Для Windows: установите Docker Desktop
```

### Проблема: Backend не подключается к БД

**Решение:**
```bash
# Проверьте PostgreSQL
docker-compose logs db

# Перезапустите БД
docker-compose restart db

# Проверьте переменную окружения
echo $SEMAPHORE_DB_URL
# Должно быть: postgres://semaphore:semaphore_pass@localhost:5432/semaphore
```

### Проблема: Демо-страница не загружается

**Решение:**
```bash
# Проверьте Nginx
docker-compose logs frontend

# Перезапустите frontend
docker-compose restart frontend

# Проверьте наличие файлов
ls -la web/public/demo-*
```

### Проблема: "Permission denied"

**Решение:**
```bash
# Сделайте скрипты исполняемыми
chmod +x demo-start.sh
chmod +x test-demo.sh
chmod +x start.sh
chmod +x stop.sh
```
